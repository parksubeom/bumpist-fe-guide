# React — 국제화 (i18n)

> **쉽게 말하면:** 화면 문구를 코드에 직접 박지 말고 언어별로 따로 관리한다.

이 스택은 **react-i18next**를 쓴다. 설정은 `src/shared/config/i18n/`.

## 문구는 하드코딩하지 않는다

```tsx
import { useTranslation } from 'react-i18next'
const { t } = useTranslation()
return <h1>{t('common.appName')}</h1>
```

- 예외: 개발 로그, 내부 식별자, 테스트 픽스처.

## 메시지 구조

- `locales/<code>/<domain>.json` — 파일명이 최상위 네임스페이스(`common.json` → `t('common.*')`).
- **`ko/`가 스키마 기준(SSOT)** — i18next 타입 보강(`Resources`)으로 키 타입 체크. `ko`에 먼저 추가 후
  다른 로케일 동기화(누락은 폴백).
- 기본 로케일 `ko`, 폴백 `en`. 초기값: `localStorage` → `navigator.language` → 기본.

## 복수형·변수·서식

- 개수 문구는 i18next pluralization, 변수는 보간(`{{count}}`), 날짜·숫자는 `Intl`.

## 전환·등록

- `i18n.changeLanguage(code)` + `localStorage` 저장 + `<html lang>` 갱신을 묶은 `setLocale` 헬퍼.
  UI는 `shared/ui`의 `LocaleSwitcher` 재사용. 인스턴스는 `app`에서 `<I18nextProvider>`로 등록.

## 테스트

- 컴포넌트 테스트는 `<I18nextProvider i18n={i18n}>`로 감싸 마운트. Storybook은 decorator로 전역 설치.
