# 36. 국제화 (i18n)

> **쉽게 말하면:** 화면 문구를 코드에 직접 박지 말고 언어별로 따로 관리한다.

이 저장소는 **vue-i18n**(Composition API 모드)을 쓴다. 설정은 `src/shared/config/i18n/`,
결정 배경은 [ADR 0011](../../../docs/adr/0011-use-vue-i18n-for-internationalization.md). 상세 사용법은
그 폴더의 `README.md` 참고.

## 문구는 하드코딩하지 않는다

- 사용자에게 보이는 모든 텍스트는 **메시지 키**로 참조한다. 템플릿에 한국어/영어 문자열을 직접 박지 않는다.
  ```vue
  <script setup lang="ts">
  import { useI18n } from 'vue-i18n'
  const { t } = useI18n() // 키는 타입 체크 + 자동완성
  </script>
  <template>
    <h1>{{ t('common.appName') }}</h1>
  </template>
  ```
- 예외: 개발용 로그, 내부 식별자, 테스트 픽스처.

## 메시지 구조 (언어별 폴더 + 도메인별 파일)

- `locales/<code>/<domain>.json` — 파일명이 곧 최상위 네임스페이스(`common.json` → `t('common.*')`,
  `nav.json` → `t('nav.*')`). 도메인이 늘면 파일을 추가하고 각 `index.ts` 배럴에 등록.
- **`ko/`가 메시지 스키마의 기준(SSOT)** — `i18n.d.ts`가 이걸로 타입을 잡는다. `ko`에 먼저 키를
  추가해야 `t()`가 인식한다. 같은 키를 다른 로케일에도 넣어 구조를 동기화(누락은 폴백으로 채워짐).
- 기본 로케일 `ko`, 폴백 `en`. 초기값: `localStorage('locale')` → `navigator.language` → 기본.

## 복수형·변수·서식

- 개수 문구는 vue-i18n pluralization을 쓰고 문자열을 직접 잇지 않는다.
- 변수는 보간(`{count}`). 문장을 조각내어 연결하면 번역이 깨진다.
- 날짜·숫자·통화는 `Intl`(또는 i18n 포맷터)로 로케일에 맞춰 출력.

## 언어 전환·등록

- 프로그램 전환은 `setLocale(code)` (활성 로케일 + `localStorage` + `<html lang>` 동시 갱신).
  UI는 `@/shared/ui`의 `LocaleSwitcher`를 재사용.
- 인스턴스는 `app/providers`에서 `app.use(i18n)`로 등록(이미 연결됨). 새 언어는
  `locales/<code>/`를 `ko/` 구조대로 만들고 루트 `index.ts`의 `SUPPORTED_LOCALES`·`LOCALE_LABELS`에 추가.

## 테스트

- 컴포넌트 유닛 테스트는 i18n 플러그인을 주입해 마운트: `mount(C, { global: { plugins: [i18n] } })`.
- Storybook은 `.storybook/preview.ts`에서 전역 설치되어 별도 설정 불필요.

## 검증

- 하드코딩된 사용자 문구가 없는지 리뷰에서 확인(`review-changes`). 로케일 전환이 라우팅에
  영향을 주면 정책을 ADR로 남긴다(`create-adr`).
